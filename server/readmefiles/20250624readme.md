# Workmates All-in-One Messenger (Backend)

올인원 협업 메신저 프로젝트의 백엔드입니다. 기술 스택은 **Node.js + Express + MySQL + Sequelize** 기반입니다.

---

## 📁 폴더 구조

```
server/
├── config/
│   └── database.js          # Sequelize 설정
├── controllers/
│   └── authController.js    # 회원가입/로그인 로직
├── models/
│   ├── user.js              # USER 테이블 모델
│   ├── email.js             # EMAIL 테이블 모델
│   └── index.js             # 모델 초기화 및 관계 설정
├── routes/
│   └── authRoutes.js        # 인증 관련 라우터
├── app.js                   # 서버 실행 엔트리 포인트
├── package.json             # 의존성 및 실행 스크립트 정의
└── .env                     # 환경 변수 파일
```

---

## 🧠 구현된 기능 요약 (1단계)

### ✅ 사용자 회원가입 `/api/auth/register`

- 이메일, 비밀번호, 닉네임을 입력받아 회원가입 처리
- 비밀번호는 bcrypt로 해싱 후 저장
- USER 테이블과 EMAIL 테이블에 데이터 저장

### ✅ 사용자 로그인 `/api/auth/login`

- 이메일과 비밀번호로 로그인 요청
- 비밀번호 비교 후 JWT 토큰 발급

## ⚙️ .env 환경 변수 설정 예시 및 설명

```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=eogh9761
DB_NAME=workmates
DB_PORT=3306
JWT_SECRET=your_jwt_secret_here
PORT=3000
```

| 변수명          | 설명                   |
| ------------ | -------------------- |
| DB\_HOST     | MySQL 호스트 주소         |
| DB\_USER     | MySQL 사용자명           |
| DB\_PASSWORD | MySQL 비밀번호           |
| DB\_NAME     | 사용할 데이터베이스 이름        |
| DB\_PORT     | MySQL 포트 (기본: 3306)  |
| JWT\_SECRET  | JWT 토큰 암호화에 사용할 키    |
| PORT         | 서버가 열릴 포트 (기본: 3000) |

## 💡 테스트 방법 요약

### 🔹 회원가입 예시 (Postman 또는 curl)

```
POST http://localhost:3000/api/auth/register
Content-Type: application/json

{
  "email": "test@example.com",
  "password": "123456",
  "nickname": "tester"
}
```

### 🔹 로그인 예시

```
POST http://localhost:3000/api/auth/login
Content-Type: application/json

{
  "email": "test@example.com",
  "password": "123456"
}
```

## ✅ 서버 실행 방법

```bash
npm install               # 의존성 설치
npx sequelize sync        # (또는 app.js 실행 시 자동 동기화)
node app.js               # 서버 실행
```
